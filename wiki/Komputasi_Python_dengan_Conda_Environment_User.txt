## 2. Membuat Conda Env dan Modul Pyload

**(1)** Buka terminal ALELEON dengan opsi:
* **Login SSH**; atau
* Login **EWS** (https://aleleonsuper.computer/), buka menu ***Clusters > Aleleon Shell Access***

**(2)** Aktifkan **modul conda** dengan pilihan:

* *Anaconda -> dibuat oleh Anaconda dan menjadi standar industri saat ini.*
* *Mamba -> "Anaconda" versi komunitas, dibuat dengan C++.*

| Package Manager Conda | Python default | Perintah Mengaktifkan |
| --- | --- | --- |
| Anaconda3 2025.06-1 | 3.13.4 | `$ ml Anaconda3/2025.06-1` |
| Anaconda3 2024.06-1 ***)** | 3.12.4 | `$ ml Anaconda3/2024.06-1` |
| Anaconda3 2024.02-1 | 3.11.7 | `$ ml Anaconda3/2024.02-1` |
| Mamba 23.11.0-0 | 3.10.13 | `$ ml Mamba/23.11.0-0` |

> **⚠️ *) Terkait Anaconda3 2024.06-1:**
> 1. *Karena banyaknya laporan bug, sementara hindari Anaconda ini.*
> 2. *Bagi user yang membuat conda env dengan Anaconda ini, disarankan:*
>    * *Hapus conda env bersangkutan.*
>    * *Buat ulang dengan versi 2025.06-1.*

**(3)** Buat **conda env** dengan format berikut.
* *Versi Python wajib didefinisikan untuk isolasi site-packages.*
* *Sesuaikan dengan kebutuhan package user.*

```bash
# --- Anaconda ---
$ conda create -n [nama-env] python=[versi-python]

# Bila tersedia file YAML resep instalasi package:
$ conda create -f [nama-file-yml].yml
```

```bash
# --- Mamba ---
$ mamba create -n [nama-env] python=[versi-python]

# Bila tersedia file YAML resep instalasi package:
$ mamba create -f [nama-file-yml].yml
```

```bash
# Contoh Anaconda, nama "skripsi", versi Python 3.12:
$ conda create -n skripsi python=3.12
```

**(4)** Aktifkan conda env dengan source:

```bash
$ source activate [nama-env]
```

Buat **modul pyload** untuk aktivasi conda env berikutnya:

```bash
(env)$ pyl mod
# 1. Pilih menu no.1
# 2. Ikuti alur script.
```

## 3. Mengaktifkan Conda Env dan Instalasi Package

**(1)** Buka terminal ALELEON dengan opsi:
* **Login SSH**; atau
* Login **EWS** (https://aleleonsuper.computer/), buka menu ***Clusters > Aleleon Shell Access***

Aktifkan conda env dengan modul pyload buatan user:

```bash
$ pyl load [nama-modul]
```

Bila lupa nama / isi modul, lihat:

```bash
# Ketersediaan modul
$ pyl list
# Lihat bagian USER-DEFINED

# Info modul
$ pyl info [nama-modul]
```

**(2)** Bila ingin menggunakan conda env di **EWS app JupyterLab**, instal package:

```bash
(env)$ pip install jupyter notebook
```

**(3)** Instal package Python kebutuhan user:
* Mendukung package manager **pip** dan **conda**.
* Secara umum mendukung semua package **CPU**.
* Perhatikan untuk package **GPU**:
  * ALELEON menggunakan GPU NVIDIA RTX 3090 (Ampere) sehingga instal **package CUDA versi >=11.**
  * Compute node GPU ALELEON terpasang 2x RTX 3090 dengan NVLink. User yang ingin menggunakan multi-GPU dapat instal package komunikasi paralel NVIDIA seperti **nvidia-nccl**.
  * Tim admin menyerahkan ke user terkait kelengkapan instalasi package NVIDIA CUDA yang dibutuhkan seperti cuDNN, cuBLAS, cuFFT, dll.

**(4)**
* Menjalankan komputasi -> subbab **5**
* Manajemen conda env dan modul pyload -> subbab **4**

## 4. Manajemen Conda Env dan Modul Pyload

Semua manajemen ini dilakukan di terminal, pilih sesuai kebutuhan:
* Permintaan tutorial manajemen lainnya **hubungi tim support**.

### Menghapus cache conda dan package

* *Conda env dan instalasi package menghasilkan file cache.*
* *Berpotensi menghabiskan storage HOME user.*
* *User dapat berkala membersihkan cache ini.*

**1.** Aktifkan modul Anaconda3 apapun:

```bash
$ ml Anaconda3
```

**2.** Hapus cache conda env:

```bash
# Menghapus index dan cache package conda.
# Tidak mempengaruhi conda env dan package di dalamnya.
$ conda clean --all

# Menghapus semua cache pip
$ pip cache purge
```

### Mengganti nama modul pyload

**1.** Buka direktori modul pyload user:

```bash
$ cd ~/.pymodules
```

**2.** Ganti nama file .sh modul.

**3.** Cek apakah nama modul sudah berubah:

```bash
$ pyl list
# Lihat bagian USER-DEFINED
```

### Menghapus conda env

**1.** Hapus conda env user:

```bash
$ ml Anaconda3
$ conda remove -n [nama-env] --all

# Bila lupa nama env, cari dengan:
$ conda env list
```

**2.** Hapus modul pyload terkait di `~/.pymodules`

## 5. Pilihan Menjalankan Komputasi

Pilih sesuai preferensi user:

* **5B** - Batch job **terminal** dengan **Slurm.**

### 5A. Batch Job dengan Job Composer EWS

Batch job berjalan di **compute node**.

**(1)**

**a.** Login ke **EWS** (https://aleleonsuper.computer/), buka menu ***Jobs > Job Composer***

**b.** Buat **ruang job** dengan menu ***New Jobs > From Template***

**(2)**

**a.** Pilih template sesuai kebutuhan:

* **Conda Environment User (CPU)**
  * *Berjalan di compute node CPU*
* **Conda Environment User (GPU)**
  * *Berjalan dengan akselerasi GPU di compute node GPU*

**b.** Isi **Job Name** *(bebas)*

**c.** klik ***[Create New Job]***

**(3)** Siapkan file komputasi via ***[Edit Files]***

* Dapat upload / download / manajemen / edit file.
* Limit upload 100MB per file.
  * Bila lebih gunakan **Aplikasi SFTP** ke direktori Job Composer.

**(4)** Pada komputasi Python, lakukan pengecekan berikut:

#### Upload file Python Notebook (ipynb)?

Batch job menjalankan file Python (`.py`).

**1.** Buka ***[Open Terminal]***

**2.** Lakukan konversi ke file Python:

```bash
$ jupytext --to py [file-ipynb-user]
```

Contoh konversi file Notebook `wololo.ipynb`:

```bash
$ jupytext --to py wololo.ipynb
```

akan menghasilkan file Python `wololo.py`

#### Apakah kelengkapan conda env user sesuai?

**1.** Buka ***[Open Terminal]***

**2.** Aktifkan modul pyload user yang akan digunakan:

```bash
$ pyl load [nama-modul]
```

**3.** Lakukan **pycheck** file Python yang akan dijalankan:

```bash
$ pycheck [file-python-user]
```

**4.** Perhatikan output:

* `READY TO GO!` -> file siap jalan.
* `MISSING PACKAGE!` -> ada kekurangan package.
  * Instal package sesuai panduan output, contoh:

```
[MISSING PACKAGE!] skripsi.py
- Import: 'usb' -> Try: pip install --user pyusb

- Jalankan:
$ pip install --user pyusb
```

**(5)** Isi kolom **submit script** untuk menjalankan job:

**a.** Klik ***[Open Editor]***

**b.** Ikuti petunjuk **NOTES** dan alur script di dalamnya.

**c.** Klik ***[Save]*** untuk menyimpan script.

*Info lengkap submit script lihat Submit Script ALELEON*

**(6)** Jalankan job dengan klik ***[Submit]***

**(7)** Pantau status dan output job:

* *Refresh halaman Job Composer untuk update status.*

| Status | Keterangan |
| --- | --- |
| **Not Submitted** | Job belum dijalankan. |
| **Queue** | Job mengantri dan belum jalan. |
| **Running** | Job jalan. User dapat memantau output dan utilisasi. |
| **Completed** | Job selesai. Bila ingin jalankan ulang job, klik **[Submit]**. |
| **Failed** | Job berhenti karena error atau distop user. |

**Saat job Running, user dapat memantau:**

* **Output job:** Lihat output di kolom **Folder Contents**. Bila ikut template, job menghasilkan:
  * `result-jobID.txt` -> output terminal
  * `error-jobID.txt` -> log status job
* **Monitor Sausage:**
  ```bash
  $ sausage
  # 1. Menu sausage: Konsumsi Core Hour
  # 2. Menu sfair: Konsumsi alokasi komputasi dan sisa limit LFU
  ```
* **Utilisasi CPU dan Memori job:**
  1. Buka homepage **EWS**
  2. Buka menu ***Monitoring > Grafana*** dan tutup *(untuk mengaktifkan pembacaan utilisasi cgroup).*
  3. Buka menu ***Jobs > Active Jobs***
  4. Ganti mode **All Jobs** ke **Your Jobs**
  5. Klik tombol detail pada job yang akan dipantau.
  6. Lihat monitor utilisasi CPU dan memori di bawah.

⚠️ Bila butuh job berhenti:
* Klik ***[Stop]***

*Bila user merasa job gagal berjalan:*
* *Job tertahan (status Queue) dalam waktu lama -> lihat Troubleshooting 1*
* *Komputasi error atau berjalan tidak semestinya -> lihat Troubleshooting 2*

Untuk download file output, gunakan opsi:
* Menu ***[Edit Files]***
* **Aplikasi SFTP** - arahkan ke direktori Job Composer.

User dapat menghapus ruang job dan datanya:
1. Pilih ruang job yang akan dihapus.
2. Klik ***[Delete]***

### 5B. Batch Job dengan Slurm di Terminal

Batch job berjalan di **compute node**.

*Panduan ini mengasumsikan user dapat menggunakan terminal Linux.*

**(1)** Buka terminal ALELEON dengan opsi:
* **Login SSH**; atau
* Login **EWS** (https://aleleonsuper.computer/), buka menu ***Clusters > Aleleon Shell Access***

**(2)** Upload / siapkan file komputasi. Opsi upload:
* **Aplikasi SFTP**
* Melalui **EWS**, menu ***Files > Home Directory***
  * *Limit upload EWS Home Directory 100MB per file.*

**(3)** Pada komputasi Python, lakukan pengecekan berikut:

#### Upload file Python Notebook (ipynb)?

Batch job menjalankan file Python (`.py`). Lakukan konversi ke file Python:

```bash
$ jupytext --to py [file-ipynb-user]

# Contoh konversi file Notebook wololo.ipynb:
$ jupytext --to py wololo.ipynb
# akan menghasilkan file Python wololo.py
```

#### Apakah kelengkapan conda env user sesuai?

**1.** Aktifkan modul pyload user yang akan digunakan:

```bash
$ pyl load [nama-modul]
```

**2.** Lakukan **pycheck** file Python yang akan dijalankan:

```bash
$ pycheck [file-python-user]
```

**3.** Perhatikan output:

* `READY TO GO!` -> file siap jalan.
* `MISSING PACKAGE!` -> ada kekurangan package.
  * Instal package sesuai panduan output.

**(4)** Buat **submit script** untuk menjalankan job:

1. Nama script bebas, format `.sh`, contoh `submit.sh`
2. Isi script dengan contoh template di bawah ini.
3. Perhatikan template ini menggunakan header `#!/bin/bash -l`
4. Ikuti petunjuk `NOTES` dan alur script di dalamnya.

**Template submit script - versi CPU:**

```bash
#!/bin/bash -l

# --------------------------------------------------
# Conda environment User (CPU)
# rev.250226
#
# NOTES:
# (1) Isi bagian yang ditandai 4 garing (////).
# (2) Ikuti petunjuk script.
# (3) Template ini bersifat referensi.
#     User dapat mengubah bagian yang perlu diubah.
# --------------------------------------------------

# ------ Spesifikasi reservasi job (SBATCH) --------
# Terdapat rentang nilai di SBATCH alokasi komputasi
# berdasarkan ketentuan metrik (SLA) ALELEON.
#
# Mengetahui rentang nilai:
# (1) Jalankan perintah:
#     $ sausage
# (2) Pilih menu slimit -> job type: Batch job
# --------------------------------------------------

# > Jumlah core thread CPU, min 2
#SBATCH --ntasks=////

# > Jumlah memori RAM (satuan GB)
#SBATCH --mem=////GB

# > Total reservasi waktu job
# > Format HH:MM:SS atau D-HH:MM:SS
# > ex, 1 hari: 24:00:00 atau 1-00:00:00
#SBATCH --time=////

# > File output terminal, %j -> ID job
#SBATCH --output=result-%j.txt

# > File output status dan error
#SBATCH --error=error-%j.txt

# --------------------------------------------------
# Script jalannya program
# --------------------------------------------------

# Muat modul pyload conda env user
pyl load ////

# Script komputasi user
////

# Cleanup
pyl unload
```

**(5)** Jalankan job dengan perintah sbatch:

```bash
$ sbatch [nama-submit-script]

# Contoh:
$ sbatch submit.sh
```

**(6)** Pantau status jalannya job dengan perintah squeue:

```bash
$ squeue -ul $USER
```

Bila tidak ada di squeue artinya job tersebut telah selesai.

| Status | Keterangan |
| --- | --- |
| **PD (PENDING)** | Job mengantri dan belum jalan. |
| **R (RUN)** | Job jalan. User dapat memantau output, sausage, dan utilisasi. |
| **CG (COMPLETING)** | Job selesai dan proses clean-up. |
| **CA (CANCELED)** | Job dibatalkan user. |
| **PR (PREEMPTED)** | Job dibatalkan oleh tim admin. Notifikasi via email support. |
| **S (SUSPENDED)** | Job ditahan oleh tim admin. Notifikasi via email support. |

**Saat job Running, user dapat memantau:**

* **Output job:** Gunakan perintah `ls`, `cat`, `more`, `tail`, dll. Bila ikut template, job menghasilkan:
  * `result-jobID.txt` -> output terminal
  * `error-jobID.txt` -> log status job
* **Monitor Sausage:**
  ```bash
  $ sausage
  # 1. Menu sausage: Konsumsi Core Hour
  # 2. Menu sfair: Konsumsi alokasi komputasi dan sisa limit LFU
  ```
* **Utilisasi CPU dan Memori job:**
  1. Login ke **EWS**
  2. Buka menu ***Monitoring > Grafana*** dan tutup *(untuk mengaktifkan pembacaan utilisasi cgroup).*
  3. Buka menu ***Jobs > Active Jobs***
  4. Ganti mode **All Jobs** ke **Your Jobs**
  5. Klik tombol detail pada job yang akan dipantau.
  6. Lihat monitor utilisasi CPU dan memori di bawah.

⚠️ Bila butuh job berhenti:

```bash
$ scancel [job-ID]

# Job ID lihat squeue diatas.
# Contoh membatalkan job ID 231:
$ scancel 231
```

*Bila user merasa job gagal berjalan:*
* *Job tertahan (state PENDING) dalam waktu lama -> lihat Troubleshooting 1*
* *Komputasi error atau berjalan tidak semestinya -> lihat Troubleshooting 2*

Untuk download file output, gunakan opsi:
* **Aplikasi SFTP**
* Melalui **EWS**, menu ***Files > Home Directory***

### 5C. Sesi Interaktif JupyterLab

Sesi Jupyter berjalan di **interactive node**.

**(1)** Login ke **EFIRO Web Service** (https://aleleonsuper.computer/)

Untuk efisiensi waktu dapat upload / siapkan file komputasi dulu via:
* Menu ***Files > Home Directory***
  * *Limit upload 100MB per file.*
* Bila lebih gunakan **Aplikasi SFTP**

**(2)** Buka menu ***Interactive Apps > Jupyter*** dan isi formulir:

* **Number of CPU threads**
* **Use GPU for the job**
  * *no -> tanpa GPU.*
  * *yes -> 1x RTX 3090.*
* **Number of hours**
  * *Reservasi waktu sesi Jupyter.*
  * *Rentang waktu ditentukan pilihan jumlah CPU, GPU, dan sisa Core Hour. Lihat dengan:*
    ```bash
    $ sausage
    # 1. Pilih menu slimit,
    # 2. Pilih job type Interactive Apps
    ```
* **Choose package to run** -> User provided
* **Choose how to launch Jupyter** -> Anaconda / Mamba
* **Choose Anaconda/Mamba version** & **User provided Conda/Mamba environment**
  * *Modul conda yang digunakan untuk membuat serta direktori conda env user.*
  * Untuk membantu pengisian:
    ```bash
    $ pyl info [nama-modul]
    ```
    Gunakan info pyload:
    * **Conda Lmod module** -> Choose Anaconda/Mamba version
    * **Conda Env PATH** -> User provided Conda/Mamba environment

User dapat save settings.

**(3)** Klik ***[Launch]***

* User dilempar ke halaman **My Interactive Sessions**
* Tunggu hingga status Running.

*Bila sesi tidak atau gagal berjalan:*
* *Status (Queued atau Starting) terlalu lama -> lihat Troubleshooting 1*
* *Status Completed walau sesi belum mulai -> lihat Troubleshooting 2*

**(4)** Bila status **Running**, klik ***[Connect to Jupyter]***

* ⚠️ Sistem mulai **menghitung Core Hour!**
* Sesi berjalan selama **waktu sesi masih ada.**
* Untuk menghentikan sesi berjalan, klik ***[Cancel]***

**(5)** Gunakan **Python 3 (ipykernel)** untuk sesi Notebook.

*Lihat Troubleshooting 2 bila ada masalah seperti:*
* *Sesi terputus sebelum waktu habis.*
* *Tidak memuat package dengan benar.*

Untuk download file output, gunakan opsi:
* Bawaan JupyterLab
* Melalui **EWS**, menu ***Files > Home Directory***
* **Aplikasi SFTP**

**Trivia dan FAQ:**

* Sesi Jupyter **tetap berjalan** selama waktu sesi masih ada, termasuk ketika:
  * Koneksi user terputus saat sesi Jupyter.
  * User keluar dari sesi Jupyter.
  * User log out dari EFIRO Web Service.
* User dapat kembali masuk ke sesi Jupyter sedang jalan via menu **My Interactive Sessions**.
* Ketika user **keluar saat ada cell jalan** dan kembali masuk:
  * Bisa jadi cell tsb. tidak menampilkan aktivitas cell.
  * Lihat status cell / output lainnya.
